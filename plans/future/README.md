# Future Feature Plans

This directory holds feature-oriented plans that are not yet part of the active implementation roadmap.

Use one Markdown file per feature. Treat each file as a working surface for comments, open questions, alternatives, product notes, and integration requirements. When a feature is ready, promote it into [../00-implementation-roadmap.md](../00-implementation-roadmap.md) as one or more reviewable chunks and update the durable reference docs it depends on.

Suggested feature-plan sections:

- Status
- Product intent
- Proposed scope
- Out of scope
- Dependencies
- Open questions and comments
- Promotion checklist
